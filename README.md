# Privacy Switch — камера, микрофон, датчики

Приложение с тремя карточками: камера, микрофон, датчики. У каждой — свой
уровень реальности, честно расписанный ниже.

## 📷 Камера — реальная блокировка

Android не даёт обычным приложениям API для отключения камеры на уровне
системы. Единственный официальный способ — зарегистрировать приложение как
**Device Admin** (администратор устройства) и использовать метод
`DevicePolicyManager.setCameraDisabled()`. Это не root и не полноценный MDM
(Device Owner), права включаются/выключаются в один клик через системный диалог.

Когда камера "выключена" через это приложение — её не сможет использовать
**ни одно** приложение на устройстве (штатная камера, WhatsApp, Zoom и т.д.),
пока вы не включите её обратно тем же переключателем.

- Кнопка **"Активировать права администратора"** — открывает системный экран
  подтверждения, где нужно нажать «Активировать».
- Переключатель камеры активен только после этого.
- Кнопка **"Деактивировать"** отзывает разрешение (камера автоматически
  возвращается в рабочее состояние).

## 🎤 Микрофон — мягкий mute (НЕ настоящая блокировка)

У Android нет публичного API для полноценного отключения микрофона обычным
приложением — `setMicrophoneMute()` из `DevicePolicyManager` доступен только
для **Device Owner** (полноценная corporate-регистрация устройства, которая
на личном телефоне требует сброса до заводских настроек и настройки через
adb/QR ещё до входа в аккаунт Google — отдельная непростая процедура).

Поэтому переключатель микрофона в приложении делает "мягкий" mute через
`AudioManager.setMicrophoneMute()`:
- работает только пока это приложение открыто;
- может быть сброшен другими приложениями, звонком, перезагрузкой;
- это не блокировка доступа, а временное приглушение текущего аудиопотока.

Приложение честно показывает это предупреждение прямо в интерфейсе.

## 📡 Датчики — только ярлык на системный экран

У Android нет публичного API, позволяющего стороннему приложению отключить
акселерометр/гироскоп/etc. во всей системе. Кнопка "Открыть системные
настройки датчиков" просто открывает системный экран приватности
(`Settings.ACTION_PRIVACY_SETTINGS`), где на многих устройствах (начиная с
Android 12) есть переключатель **"Датчики выкл."** — включить его можно
только вручную, руками пользователя, программно это сделать нельзя.

## Сборка

1. Откройте папку `CameraToggleApp` в Android Studio (Meerkat/2024.x или новее).
2. Дождитесь синхронизации Gradle (нужен установленный Android SDK 36 —
   Android 16; если его ещё нет в вашей Android Studio, откройте
   `app/build.gradle.kts` и временно поставьте `compileSdk = 35`,
   `targetSdk = 35` — на функциональность это не влияет).
3. Запустите на устройстве/эмуляторе с Android 15 или 16 (minSdk 26, так что
   заработает и на более старых версиях тоже).

## Ограничения, о которых стоит знать

- Это блокировка **на уровне всего устройства**, а не выборочно для одного
  приложения — Android не позволяет обычным приложениям выборочно закрывать
  камеру только конкретным программам.
- Некоторые производители (Xiaomi, Samsung и т.п.) могут показывать
  дополнительные предупреждения при активации "администратора устройства" —
  это нормально, соглашайтесь в системном диалоге.
- Если на устройстве уже есть корпоративный профиль (Work Profile) или другой
  MDM, политики могут конфликтовать — в этом случае разработка потребует
  отдельного тестирования.
